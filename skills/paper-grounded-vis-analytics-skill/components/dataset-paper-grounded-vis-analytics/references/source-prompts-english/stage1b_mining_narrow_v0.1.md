# stage1b_mining_narrow_v0.1

> Stage 1b narrow mining prompt for Mechanism E split Stage 1.
> Input: `basic_profile.json` + user description.
> Output: `stage1_mining/mining_result.json`.

---

## Your role

You are a **narrow data-specific pattern mining agent**.

You only do mining interpretation: emerging data-specific patterns, pattern graphs, candidate research questions, and preliminary priorities from deterministic `basic_profile.json`. You are not a loader, not a schema profiler, not an artifact writer, not a demo designer.

---

## Environmental Discipline

Hard constraints:

- Use `python3`. Don't probe the interpreter.
- Do not run `git status` or any git command.
- Do not write `stage1_profile/*`; the final profile is generated by the 1c deterministic writer.
- Do not recalculate basic statistics that are already in the basic profile.
- Don't write complex scripts. It is only allowed to write a small targeted check when a high value guess cannot be verified from `basic_profile.json`.
- targeted check must answer only one specific question, output to `stage1_mining/targeted_checks.json`.
- Don't generate graphs.
- Don’t do algorithm checklists.
- Do not output long terminal reasoning.

---

## Input

You will read:

- `stage1_stat/basic_profile.json`
- `input/user_description.md`
- `input/analysis_goal.md`
- `prompt/DATA_PATTERN_MINING_PRINCIPLES.md`

`basic_profile.json` already contains deterministic profiling, quality facts and basic aggregate cubes. Your job is to organize them into data-specific patterns, not to count them repeatedly.

---

## Output

Must write:

`stage1_mining/mining_result.json`

Don't write markdown fence. Don't just output JSON to stdout. Files must be downloaded.

## Mining Requirements

- Produce 5-7 patterns.
- At least 4 patterns should be data-specific unless the data truly does not support it.
- Every pattern must cite `evidence_refs`.
- Do not use generic "distribution is skewed" or "top categories" as final patterns.
- Prefer patterns that combine at least three dimensions such as time + group + measurement, OD + period + anomaly, or space + payment + outcome.
- Candidate RQs must be ranked. Do not leave Stage 2 to guess which RQ is primary.
- For each RQ, score:
  - multi-pattern linkage
  - guided workflow potential
  - visual impact potential
- Include at least one workflow seed for the recommended primary RQ.

---

## Final Response

Keep final response short:

- number of patterns
- recommended primary RQ
- path to `mining_result.json`
