# Retrieval And Reference Learning

## Compatible Paper Library

This skill expects the paper library generated by `$paper-mineru-scholar-index`:

```text
data/index.db
data/faiss.index
data/faiss_ids.json
data/papers/<dir_name>/meta.json
data/papers/<dir_name>/paper.md
```

Layer mapping:

- L1: `meta.title`
- L2: `meta.abstract`
- L3: `meta.l3_conclusion`
- L4: `paper.md`

The paper-level vector should represent L1+L2+L3.

## Retrieval Strategy

Use hybrid retrieval when possible:

1. Extract a query from the input paper/dataset.
2. Run FTS5 over `papers` in `data/index.db`.
3. If an embedding endpoint is available, embed the query with `qwen3-0.6B-embedding` and compare against `paper_vectors`.
4. Fuse scores by rank, then read local `meta.json` and `paper.md` for top papers.

If embeddings are unavailable, FTS5 retrieval is acceptable for feasibility runs. Mark retrieval mode as `fts_only`.

## Borrowed Element Rules

Borrowed elements must be specific and auditable:

- Good: "Adapt linked brushing between a semantic overview and an evidence panel, supported by the paper's summary of interactive querying."
- Bad: "Inspired by VIS literature."
- Bad: "Use a chord diagram" unless the reference actually discusses flow, relation, or chord-like encodings.

Each borrowed element should include:

- `source_paper_id`
- `source_title`
- `evidence_field`: title, abstract, l3_conclusion, or paper_snippet
- `borrowed_element`
- `adapted_for_current_input`
- `mapped_to_view_ids`
- `mapped_to_interaction_ids`
- `confidence`

## Fallback Standard Basis

When retrieval is weak, use fallback VIS basis openly:

- overview + detail
- focus + context
- linked brushing
- time/phase strip + structure view
- spatial/context view + relation view
- aggregate view + raw evidence trace
- anomaly strip as secondary evidence, not primary unless selected by the input target

Do not pretend fallback basis is a paper precedent.
